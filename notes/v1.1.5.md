## v1.1.5 Changelog

Three targeted fixes: markdown rendering in the session log, a long hang at the end of Claude generations, and the flashing Windows console windows.

### Improvements
- Session console now renders Markdown in AI output (**bold**, *italic*, `inline code`, code blocks, headings, lists, links, blockquotes) instead of showing the raw markup
- Windows launch and runtime switching no longer pop up any black console windows; startup also feels noticeably snappier as a side effect

### Fixes
- Fixed a long UI freeze (10s+) right before the final design renders when using Claude — the new Claude CLI emits a `stream_event` for every text delta, which for a design-sized write turns into thousands of events per turn; we used to append each one to the session output string, causing O(n²) string copies and reactive re-renders that locked the main thread. These deltas are now filtered since their content is already aggregated in the assistant/result events that follow
- Reduced the chance of Windows launch failures caused by antivirus heuristics reacting to the rapid-fire console-window flashes on startup

### How to Use
- **Mac (Apple Silicon)** — Download the `aarch64.dmg` file
- **Windows** — Download the `x64-setup.exe` file
